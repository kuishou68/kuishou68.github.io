12:T1365,<h3 id="1业务场景">1、业务场景</h3>
<p>阅读之前，想了解一下各位观众老爷们，你们公司的项目是怎么部署的：</p>
<ul>
<li>1.本地打包手动上传服务器；</li>
<li>2.本地打包自动上传服务器；</li>
<li>3.代码仓库流水线自动构建；</li>
<li>4.其他...；</li>
</ul>
<p>我们用的第3种部署方式，仓库代码合并到release之后自动打包部署，不过这个过程很耗费时间，且流水线是要收钱的。有一天，流水线到期了，部门主管打算节约成本，于是就开始让我们自己写一个脚本，用来部署公司项目；</p>
<p>前端静态资源(项目图片、icon、打包css、js)文件放在七牛，域名是阿里的OSS，这种情况和大多数部署方式不同，你们一般直接把所有静态资源放在服务器就行，然后用域名指向服务器ip;
但是对于公司有很多个项目，每个项目都有自己的域名，数据库、redis等的时候，服务器的压力就会很大。如果把服务器的静态资源分出去，且部署在OSS上，对服务器的压力会相对少很多，说干就干；</p>
<h3 id="2具体实现">2、具体实现</h3>
<pre><code>npm install ssh2-sftp-client</code></pre>

<p>sftp.prod.js 主要用于将打包后的 index.html 文件上传到阿里OSS；</p>
<pre><code>// sftp.prod.js
const path = require(&quot;path&quot;);
// eslint-disable-next-line no-undef
const SftpClient = require(&quot;ssh2-sftp-client&quot;);

const sftp = new SftpClient();

let options = {
    port:&quot;&quot;,
    host: &quot;&quot;, // sftp地址
    username:&quot;&quot;,
    password:&quot;&quot;,
    localPath:path.join(__dirname, &quot;dist/index.html&quot;)
};

const { port, host, username, password} = options;
sftp.connect({
    host,
    port,
    username,
    password
}).then(async() =&gt; {
    await sftp.put(options.localPath, &quot;/html/index.html&quot;);
    console.log(&quot;str===&gt;&gt;&gt;&gt;上传结束&quot;);
    return Promise.resolve();
})
    .then((res)=&gt;{
       sftp.end();
    })
    .catch((err) =&gt; {
       console.log(err);
       sftp.end();
    });</code></pre>
