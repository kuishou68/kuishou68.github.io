<h2 id="1-什么是-gemini-cli">1. 什么是 Gemini CLI？</h2>
<h3 id="11-简单来说">1.1 简单来说</h3>
<p>Gemini CLI 是 Google 开源的一个 <strong>AI 命令行代理</strong>（AI
Agent）。你可以把它想象成一个住在你终端里的智能助手，它能：</p>
<ul>
<li><strong>理解你的自然语言指令</strong>：比如"帮我修复这个
bug"、"给这个函数加上单元测试"</li>
<li><strong>自动执行操作</strong>：读写文件、运行命令、搜索代码</li>
<li><strong>记住你的偏好</strong>：你喜欢用 TypeScript、偏好 4
空格缩进等</li>
<li><strong>持续对话</strong>：可以恢复之前的会话继续工作</li>
</ul>

<h3 id="22-核心设计原则">2.2 核心设计原则</h3>
<p>项目采用了几个关键的设计原则：</p>
<p><strong>1. 前后端分离</strong></p>
<ul>
<li><code>packages/cli</code>：负责用户界面、输入处理、显示渲染</li>
<li><code>packages/core</code>：负责业务逻辑、API 调用、工具执行</li>
</ul>
<p><strong>2. 流式处理</strong></p>
<ul>
<li>使用 JavaScript 的 <code>AsyncGenerator</code> 实现流式响应</li>
<li>用户可以实时看到 AI 的输出，而不是等待完整响应</li>
</ul>

<li>外部工具通过 MCP 协议动态发现和调用</li>
</ul>
<p><strong>4. 分层记忆</strong></p>
<ul>
<li>全局记忆：用户级别的偏好设置</li>
<li>项目记忆：项目级别的约定</li>
<li>会话记忆：当前对话的上下文</li>
</ul>

<h3 id="51-token-的概念">5.1 Token 的概念</h3>
<p>在与 AI 对话时，每条消息都会消耗 "Token"。Token 大致可以理解为：</p>
<ul>
<li>英文：大约 4 个字符 = 1 个 Token</li>
<li>中文：大约 1-2 个字符 = 1 个 Token</li>
</ul>
<p>每个模型都有 Token 限制：</p>
<div class="sourceCode" id="cb12"><pre
class="sourceCode typescript"><code class="sourceCode typescript"><span id="cb12-1"><a href="#cb12-1" aria-hidden="true" tabindex="-1"></a><span class="co">// packages/core/src/core/tokenLimits.ts</span></span>

<h3 id="61-工具的概念">6.1 工具的概念</h3>

需要读文件、执行命令时，它会"调用工具"：</p>
<pre class="mermaid"><code>sequenceDiagram
    participant AI as Gemini AI
    participant Scheduler as ToolScheduler
    participant Tool as 具体工具
    participant FS as 文件系统/Shell

    AI-&gt;&gt;Scheduler: 请求调用 read_file
    Scheduler-&gt;&gt;Scheduler: 验证参数
    Scheduler-&gt;&gt;Tool: 创建 ReadFileTool 实例
    Tool-&gt;&gt;Tool: shouldConfirmExecute()

    alt 需要确认
        Tool--&gt;&gt;User: 显示确认对话框
        User--&gt;&gt;Tool: 确认执行
    end

    Tool-&gt;&gt;FS: 读取文件
    FS--&gt;&gt;Tool: 文件内容
    Tool--&gt;&gt;Scheduler: 返回结果
    Scheduler--&gt;&gt;AI: 工具调用结果</code></pre>

<h3 id="71-什么是-mcp">7.1 什么是 MCP？</h3>
<p><a href="https://juejin.cn/post/7555815079645036598">MCP</a>（Model
Context Protocol）是一个<strong>开放协议</strong>，用于标准化 AI
模型与外部工具之间的通信。就像 USB 让各种设备能够连接电脑一样，MCP
让各种工具能够接入 AI。</p>

        E[自定义工具&lt;br/&gt;任何功能]
    end

<h3 id="81-与-gemini-api-的通信">8.1 与 Gemini API 的通信</h3>
<p>Gemini CLI 使用 <strong>REST + SSE</strong> 的混合通信方式：</p>
<pre class="mermaid"><code>sequenceDiagram
    participant CLI as Gemini CLI
    participant API as Gemini API

    CLI-&gt;&gt;API: POST /generateContent&lt;br/&gt;(包含历史、工具、配置)

    Note over API: 开始生成响应

    loop 流式响应 (SSE)
        API--&gt;&gt;CLI: data: {&amp;#34;text&amp;#34;: &amp;#34;正在&amp;#34;}
        API--&gt;&gt;CLI: data: {&amp;#34;text&amp;#34;: &amp;#34;思考&amp;#34;}
        API--&gt;&gt;CLI: data: {&amp;#34;text&amp;#34;: &amp;#34;...&amp;#34;}
    end

    alt 需要调用工具
        API--&gt;&gt;CLI: data: {&amp;#34;functionCall&amp;#34;: {...}}
        CLI-&gt;&gt;CLI: 执行工具

<h3 id="91-hook-的概念">9.1 Hook 的概念</h3>
<p>Hook 允许你在 AI 执行的关键节点插入自定义逻辑，比如：</p>
<ul>
<li>在工具执行前审批</li>
<li>在模型调用前/后记录日志</li>
<li>在会话开始/结束时执行清理</li>
</ul>
