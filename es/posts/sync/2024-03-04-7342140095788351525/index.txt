</ul>

<h3 id="二什么是拆包分包">二、什么是拆包/分包？</h3>
<ul>
<li>2.1
分包（code-splitting）是一种优化技术，用于将应用程序的代码拆分成多个较小的包（或块）。这种技术的目的是减少初始加载时需要下载的代码量，以提高应用程序的性能和加载速度。我们这里用的Vite，Vite底层本身用的rollupjs，这边重点阐述Vite的拆包方式。在使用
build.rollupOptions.output.manualChunks 时，<a
href="https://link.juejin.cn?target=https%3A%2F%2Fcn.rollupjs.org%2Fconfiguration-options%2F%23output-manualchunks"
title="https://cn.rollupjs.org/configuration-options/#output-manualchunks">Rollup官方拆包方式</a></li>
<li>2.2 该选项允许你创建自定义的公共
chunk。当值为对象形式时，每个属性代表一个
chunk，其中包含列出的模块及其所有依赖，除非他们已经在其他 chunk
中，否则将会是模块图（module graph）的一部分。chunk
的名称由对象属性的键决定。</li>
</ul>
<pre><code>manualChunks: {  
    lodash: [&#39;lodash&#39;]  
}</code></pre>
<ul>
<li>2.3 上述例子中，即使你只是使用 import get from 'lodash/get'
形式引入，Rollup 也会将 lodash 的所有模块放到一个自定义 chunk 中。</li>
<li>2.4
当该选项值为函数形式时，每个被解析的模块都会经过该函数处理。如果函数返回字符串，那么该模块及其所有依赖将被添加到以返回字符串命名的自定义
chunk 中。例如，以下例子会创建一个命名为 vendor 的 chunk，它包含所有在
node_modules 中的依赖：</li>
</ul>
<pre><code>function manualChunks(id) {  
    if (id.includes(&#39;node_modules&#39;)) {  
        return &#39;vendor&#39;;  
    }  
}</code></pre>
<ul>
<li>安装包大小分析插件，来看下你项目中那些内容占了主要的空间；<br />
<code>npm install --save-dev rollup-plugin-visualizer</code><br />
或者<br />
<code>yarn add --dev rollup-plugin-visualizer</code></li>
<li>vite.config.js 中 引入</li>
</ul>
<pre><code>import { visualizer } from &quot;rollup-plugin-visualizer&quot;;  
module.exports = {  
    plugins: [  
        visualizer({  
            open: true, // 注意这里要设置为true，否则无效  
            gzipSize: true, // 分析图生成的文件名  
            brotliSize: true, // 收集 brotli 大小并将其显示  
            filename: &quot;stats.html&quot;, // 分析图生成的文件名  
        }),  
    ],  
};</code></pre>
<ul>
<li>2.5 配置完执行一下 <code>npm run build</code>, 项目根目录会出现一个
<code>stats.html</code> 的文件,
用浏览器打开可以看到，目前暂用整个项目体积最大的是echarts和ant-design-vue,接下来就是想办法把他们单独拆成一个独立的文件，并且上传到CDN上面，通过配置baseUrl,
打包结果中的 index.html 只需引入这些独立的文件即可； <img
src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ecd917099c934c75999e04d9e8e028e3~tplv-k3u1fbpfcp-jj-mark:3024:0:0:0:q75.awebp#?w=3364&amp;h=1560&amp;s=5325341&amp;e=png&amp;b=955e5c"
alt="image.png" /></li>
</ul>
<h3 id="三vite如何拆包分包">三、Vite如何拆包/分包？</h3>
<ul>
<li>3.1 如果是Vite
2.9以下版本，那你恭喜你，用上面Rollup官方文档配置方式即可，但是请看<a
href="https://link.juejin.cn?target=https%3A%2F%2Fvitejs.cn%2Fvite3-cn%2Fguide%2Fbuild.html%23chunking-strategy"
title="https://vitejs.cn/vite3-cn/guide/build.html#chunking-strategy">官方文档</a><br />
<img
src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ca7fe4556ffb470f83e061a10d599813~tplv-k3u1fbpfcp-jj-mark:3024:0:0:0:q75.awebp#?w=1920&amp;h=945&amp;s=140789&amp;e=jpg&amp;b=fcfcfc"
alt="image.png" /></li>
<li>3.2 到 Vite2.8 时，默认的策略是将 chunk 分割为 index 和
vendor。这对一些 SPA
来说是好的策略，但是要对每一种用例目标都提供一种通用解决方案是非常困难的。从
Vite2.9 起，manualChunks
默认情况下不再被更改。你可以通过在配置文件中添加 splitVendorChunkPlugin
