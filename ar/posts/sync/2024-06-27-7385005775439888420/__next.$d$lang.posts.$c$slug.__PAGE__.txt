<h4
id="浏览器渲染主线程与合成线程都做了什么">浏览器渲染主线程与合成线程都做了什么？</h4>
<p>在现代浏览器中，渲染过程不仅仅依赖于单一的主线程，合成线程也发挥着重要作用。理解主线程和合成线程的分工与协作有助于我们更好地优化前端性能和用户体验。</p>
<h3 id="一浏览器渲染主线程">一、浏览器渲染主线程</h3>
<p>渲染主线程是浏览器处理大部分渲染任务的核心线程。它负责以下关键任务：</p>
<p><img
src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f6d08a9bdfb340598ddaa2d4691542cc~tplv-k3u1fbpfcp-jj-mark:3024:0:0:0:q75.awebp#?w=663&amp;h=307&amp;s=124807&amp;e=png&amp;b=fdfbfb"
loading="lazy" alt="image.png" /></p>
<h4 id="11-解析和构建dom树">1.1 解析和构建DOM树</h4>
<p>当浏览器接收到HTML文档时，主线程开始解析HTML并构建DOM树。它将HTML标签、属性和内容解析成DOM节点，形成一个树状结构。</p>
<h4 id="12-解析和构建cssom树">1.2 解析和构建CSSOM树</h4>
<p>主线程同时解析CSS文件，并将其转换为CSSOM树。CSSOM树表示了所有样式规则及其层级关系。</p>
<h4 id="13-执行javascript">1.3 执行JavaScript</h4>
<p>JavaScript的执行也是在主线程上进行的。JavaScript可以直接操作DOM和CSSOM，这意味着它可以修改页面结构和样式。由于JavaScript的执行可能会阻塞HTML解析，现代浏览器采用了异步执行策略（如<code>async</code>和<code>defer</code>）来优化性能。</p>
<h4 id="14-样式计算">1.4 样式计算</h4>
<p>样式计算是指将CSSOM树中的样式规则应用到DOM树中的每个元素，计算出每个元素的最终样式。这个过程也称为“样式应用”（Style
Application）或“样式解决”（Style Resolution）。</p>
<p>样式计算的步骤如下：</p>
<ol type="1">
<li><strong>继承和层叠</strong>：确定哪些样式是继承的，以及哪些样式规则在层叠样式表（CSS
Cascading）中优先级更高。</li>
<li><strong>计算具体值</strong>：将相对值（如<code>em</code>、<code>%</code>等）转换为绝对值（如<code>px</code>）。</li>
</ol>
<h4 id="15-分层">1.5 分层</h4>
<p>在样式计算完成后，浏览器会根据特定规则将页面内容分割成多个图层。这一步骤在主线程上进行，称为“分层”（Layering）。</p>
<p>分层的目的是为了优化页面渲染性能，特别是处理复杂的页面布局和动画时。常见的触发分层的条件包括：</p>
<ul>
<li>元素使用<code>position: fixed</code>或<code>position: sticky</code>。</li>
<li>元素使用CSS属性<code>transform</code>、<code>opacity</code>、<code>filter</code>等。</li>
<li>元素包含动画或3D变换。</li>
</ul>
<h4 id="16-生成渲染树">1.6 生成渲染树</h4>
<p>主线程将DOM树和CSSOM树结合生成渲染树。渲染树包含了所有可见的DOM节点及其样式信息，用于后续的布局和绘制。</p>
<h4 id="17-布局和绘制">1.7 布局和绘制</h4>
<p>主线程负责布局和绘制操作：</p>
<ul>
<li><strong>布局</strong>：确定每个渲染对象的位置和大小。</li>
<li><strong>绘制</strong>：将每个渲染对象转换为屏幕上的像素。</li>
</ul>
<h3 id="二合成线程">二、合成线程</h3>
<p>合成线程在现代浏览器中引入，主要为了提高渲染性能，特别是提升页面滚动和动画的流畅度。它与主线程协同工作，但专注于特定的渲染任务。</p>
<h4 id="21-图层的生成和管理">2.1 图层的生成和管理</h4>
<p>合成线程负责生成和管理图层（Layers）。当页面包含复杂的样式或动画时，浏览器会将不同部分的内容分配到不同的图层。</p>
<h4 id="22-分块tiling">2.2 分块（Tiling）</h4>
<p>为了更高效地管理和渲染图层，合成线程会将图层分割成多个小块（Tiles）。每个小块的大小通常为256x256或512x512像素。这些小块便于并行处理和增量更新。</p>
<h4 id="23-光栅化rasterization">2.3 光栅化（Rasterization）</h4>
<p>分块后，每个小块需要进行光栅化，将矢量图形转换为位图。光栅化通常在GPU上完成，以提高性能。这一步骤将矢量图形信息（如路径、颜色、渐变等）转换成具体的像素数据。</p>
<h4 id="24-合成和绘制">2.4 合成和绘制</h4>
<p>合成线程将各个图层合成（Composite）到一起，并将其绘制到屏幕上。合成操作在GPU上执行，可以极大地提升性能，特别是在处理滚动和动画时。</p>
<h3 id="三主线程与合成线程的协作">三、主线程与合成线程的协作</h3>
<p>主线程和合成线程协作完成页面的渲染任务。以下是两者协作的关键点：</p>
<h4 id="31-图层的更新">3.1 图层的更新</h4>
<p>当主线程检测到需要更新图层时，会通知合成线程。合成线程接收到更新信息后，会重新合成图层并绘制到屏幕上。</p>
<h4 id="32-图层的同步">3.2 图层的同步</h4>
<p>主线程和合成线程之间需要频繁同步图层信息，确保合成线程能够准确绘制最新的页面状态。浏览器通过“帧请求”（Frame
Request）的机制实现这种同步。</p>
<h4 id="33-最小化主线程阻塞">3.3 最小化主线程阻塞</h4>
<p>为了优化性能，减少主线程的阻塞时间非常重要。开发者可以通过以下策略实现：</p>
<ul>
<li><strong>异步加载资源</strong>：使用<code>async</code>和<code>defer</code>属性异步加载JavaScript文件。</li>
<li><strong>减少重绘和回流</strong>：批量操作DOM，避免频繁触发重绘和回流。</li>
<li><strong>利用CSS动画</strong>：将动画和过渡效果交给CSS处理，而不是使用JavaScript动画。</li>
</ul>
<h3 id="四性能优化策略">四、性能优化策略</h3>
<p>理解主线程和合成线程的工作机制后，开发者可以采取以下优化策略：</p>
<h4 id="41-优化布局和绘制">4.1 优化布局和绘制</h4>
<ul>
<li><strong>减少DOM复杂度</strong>：优化HTML结构，减少DOM节点数量。</li>
<li><strong>合并样式计算</strong>：将多个样式修改合并为一次操作，减少样式计算次数。</li>
<li><strong>避免大范围重排</strong>：尽量避免触发大范围的布局重排（Reflow）。</li>
</ul>
<h4 id="42-利用合成线程">4.2 利用合成线程</h4>
<ul>
<li><strong>使用<code>will-change</code></strong>：提示浏览器哪些属性会发生变化，提前优化处理。</li>
<li><strong>利用GPU加速</strong>：使用CSS属性<code>transform</code>、<code>opacity</code>等，利用GPU提升渲染性能。</li>
</ul>
<h4 id="43-优化javascript执行">4.3 优化JavaScript执行</h4>
<ul>
<li><strong>异步加载脚本</strong>：使用<code>async</code>和<code>defer</code>属性异步加载JavaScript文件，减少阻塞。</li>
<li><strong>减少计算量</strong>：优化JavaScript代码，减少复杂计算和频繁的DOM操作。</li>
<li><strong>避免长任务</strong>：将长时间执行的任务分解为多个小任务，避免阻塞主线程。</li>
</ul>

<p>浏览器渲染主线程和合成线程在现代浏览器中扮演着重要角色，它们协作完成页面的解析、样式计算、分层、布局、绘制和合成操作。理解这两者的工作机制有助于前端开发者优化页面性能，提高用户体验。</p>
