<p>在 Vue 中，<code>watch</code>
主要用于监听响应式数据的变化并执行回调函数。根据使用方式和场景的不同，<code>watch</code>
可以分为以下几种类型：</p>
<h3 id="1-普通-watch">1. <strong>普通 <code>watch</code></strong></h3>
<p>通过 <code>watch</code> 选项或 <code>this.$watch</code>
方法定义的监听器，用于监听单个响应式数据的变化。</p>

<h3 id="总结">总结</h3>

<ol type="1">
<li>普通 <code>watch</code></li>

<li>立即执行 (<code>immediate: true</code>)</li>
<li>监听多个数据</li>

<li>监听路由变化</li>
<li>监听 <code>props</code> 的变化</li>
</ol>
<p>每种 <code>watch</code>
都有其特定的使用场景，开发者可以根据需求选择合适的监听方式。</p>

<h3 id="1-watcher-类">1. <strong>Watcher 类</strong></h3>
<p>在 Vue 源码中，<code>watch</code> 的核心是 <strong>Watcher
类</strong>。Watcher
的作用是监听一个表达式（或函数），并在其依赖的响应式数据发生变化时执行回调。</p>
<h4 id="源码位置">源码位置：</h4>
<ul>
<li><code>src/core/observer/watcher.ts</code>（Vue 2.x）</li>
<li><code>src/core/observer/watcher.js</code>（Vue 2.x）</li>
</ul>
<h4 id="关键属性">关键属性：</h4>
<ul>
<li><code>getter</code>：用于获取监听的值（可以是一个函数或表达式）。</li>
<li><code>cb</code>：回调函数，当监听的值发生变化时执行。</li>
<li><code>deps</code>：依赖列表，存储当前 Watcher 依赖的所有 Dep
实例。</li>
<li><code>deep</code>：是否深度监听。</li>
<li><code>immediate</code>：是否立即执行回调。</li>
</ul>
<h4 id="核心逻辑">核心逻辑：</h4>
<ul>
<li>Watcher 在初始化时会调用 <code>this.get()</code>，通过
<code>getter</code> 获取当前值，并触发依赖收集。</li>
<li>当依赖的响应式数据发生变化时，Watcher 会重新执行
<code>getter</code>，并调用回调函数 <code>cb</code>。</li>
</ul>
<h3 id="2-dep-类">2. <strong>Dep 类</strong></h3>
<p><code>Dep</code> 是 Vue
响应式系统中的依赖管理器，每个响应式属性都会有一个对应的
<code>Dep</code> 实例。Watcher 和 Dep 的关系是多对多的：</p>
<ul>
<li>一个 Watcher 可以依赖多个 Dep（监听多个响应式数据）。</li>
<li>一个 Dep 可以被多个 Watcher 订阅（多个地方监听同一个数据）。</li>
</ul>
<h4 id="源码位置-1">源码位置：</h4>
<ul>
<li><code>src/core/observer/dep.ts</code>（Vue 2.x）</li>
<li><code>src/core/observer/dep.js</code>（Vue 2.x）</li>
</ul>
<h4 id="核心逻辑-1">核心逻辑：</h4>
<ul>
<li>当 Watcher 调用 <code>getter</code> 时，会触发响应式数据的
<code>get</code> 拦截器，将当前 Watcher 添加到 Dep 的订阅列表中。</li>
<li>当响应式数据发生变化时，会触发 <code>set</code> 拦截器，通知 Dep
中的所有 Watcher 更新。</li>
</ul>
<h3 id="3-watch-的实现">3. <strong>watch 的实现</strong></h3>
<p>在 Vue 中，<code>watch</code> 的实现是通过创建一个 Watcher
实例来完成的。具体步骤如下：</p>
<h4 id="源码位置-2">源码位置：</h4>
<ul>
<li><code>src/core/instance/state.ts</code>（Vue 2.x）</li>
<li><code>src/core/instance/state.js</code>（Vue 2.x）</li>
</ul>
<h4 id="核心逻辑-2">核心逻辑：</h4>
<ol type="1">
<li><p><strong>初始化 <code>watch</code></strong>：</p>
<ul>
<li>在组件初始化时，Vue 会遍历 <code>watch</code>
选项，为每个监听属性创建一个 Watcher 实例。</li>
<li>例如：
<div class="sourceCode" id="cb8"><pre
class="sourceCode javascript"><code class="sourceCode javascript"><span id="cb8-1"><a href="#cb8-1" aria-hidden="true" tabindex="-1"></a>watch<span class="op">:</span> {</span>
<span id="cb8-2"><a href="#cb8-2" aria-hidden="true" tabindex="-1"></a>  <span class="fu">message</span>(newVal<span class="op">,</span> oldVal) {</span>
<span id="cb8-3"><a href="#cb8-3" aria-hidden="true" tabindex="-1"></a>    <span class="bu">console</span><span class="op">.</span><span class="fu">log</span>(<span class="st">&#39;message 发生变化:&#39;</span><span class="op">,</span> newVal<span class="op">,</span> oldVal)<span class="op">;</span></span>

会创建一个 Watcher，监听 <code>this.message</code> 的变化。</li>
</ul></li>
<li><p><strong>创建 Watcher</strong>：</p>
<ul>
<li>对于每个监听属性，Vue 会调用 <code>createWatcher</code>
方法，最终创建一个 Watcher 实例。</li>
<li>Watcher 的 <code>getter</code> 是一个函数，用于获取监听的值。</li>
<li>Watcher 的 <code>cb</code> 是用户定义的回调函数。</li>
</ul></li>
<li><p><strong>依赖收集</strong>：</p>
<ul>
<li>当 Watcher 首次执行时，会调用 <code>getter</code>，触发响应式数据的
<code>get</code> 拦截器，将当前 Watcher 添加到 Dep 的订阅列表中。</li>
</ul></li>
<li><p><strong>触发更新</strong>：</p>
<ul>
<li>当响应式数据发生变化时，会触发 <code>set</code> 拦截器，通知 Dep
中的所有 Watcher 执行更新。</li>
<li>Watcher 会重新调用 <code>getter</code> 获取新值，并执行回调函数
<code>cb</code>。</li>
</ul></li>
</ol>
<h3 id="4-深度监听的实现">4. <strong>深度监听的实现</strong></h3>
<p>深度监听是通过递归遍历对象或数组，为每个子属性创建 Watcher
来实现的。</p>
<h4 id="核心逻辑-3">核心逻辑：</h4>
<ul>
<li>当 <code>deep: true</code> 时，Watcher 会在 <code>getter</code>
中递归遍历对象或数组的所有属性。</li>
<li>每个子属性都会触发 <code>get</code> 拦截器，将当前 Watcher
添加到对应的 Dep 中。</li>
<li>这样，当对象或数组的任意子属性发生变化时，都会触发 Watcher
的更新。</li>
</ul>

<span id="cb9-2"><a href="#cb9-2" aria-hidden="true" tabindex="-1"></a>  <span class="fu">traverse</span>(value)<span class="op">;</span> <span class="co">// 递归遍历对象或数组</span></span>

<h3 id="5-immediate-的实现">5. <strong>immediate 的实现</strong></h3>
<p><code>immediate: true</code> 的实现非常简单，就是在 Watcher
创建时立即执行一次回调。</p>
<h4 id="核心逻辑-4">核心逻辑：</h4>
<ul>
<li>如果 <code>immediate</code> 为 <code>true</code>，则在 Watcher
初始化时直接调用 <code>cb</code>。</li>
</ul>

(<code>$watch</code>) 的实现</strong></h3>
<p><code>$watch</code> 是 Vue 实例的一个方法，用于动态创建 Watcher。</p>
<h4 id="核心逻辑-5">核心逻辑：</h4>

<code>createWatcher</code>，最终创建一个 Watcher 实例。</li>
<li>返回一个取消监听的函数，用于移除 Watcher。</li>

<ol type="1">
<li><strong>Watcher</strong>：负责监听数据变化并执行回调。</li>
<li><strong>Dep</strong>：负责管理依赖关系，通知 Watcher 更新。</li>
<li><strong>深度监听</strong>：通过递归遍历对象或数组实现。</li>
<li><strong>immediate</strong>：在 Watcher 初始化时立即执行回调。</li>
<li><strong>动态监听</strong>：通过 <code>$watch</code> 方法动态创建
Watcher。这些机制共同构成了 Vue 强大的响应式系统，使得
<code>watch</code> 能够高效地监听数据变化并执行相应的逻辑。</li>
</ol>

<h3 id="1-响应式数据的依赖收集data-watcher">1.
<strong>响应式数据的依赖收集（Data Watcher）</strong></h3>
<p>在 Vue 中，<code>data</code> 中定义的响应式数据会通过
<code>Object.defineProperty</code>（Vue 2.x）或 <code>Proxy</code>（Vue
3.x）进行劫持，并为每个属性创建一个 <strong>Dep</strong>
实例。当这些数据被访问时，Vue 会通过 <strong>Watcher</strong>
进行依赖收集。</p>
<h4 id="源码实现">源码实现：</h4>
<ul>
<li><p><strong>Vue 2.x</strong>：</p>
<ul>
<li>在 <code>src/core/observer/index.js</code>
中，<code>defineReactive</code> 函数会为每个属性创建一个
<code>Dep</code> 实例。</li>
<li>当数据被访问时，会触发 <code>getter</code>，将当前的
<code>Watcher</code>（如果有）添加到 <code>Dep</code>
的订阅列表中。</li>
<li>当数据被修改时，会触发 <code>setter</code>，通知 <code>Dep</code>
中的所有 <code>Watcher</code> 更新。</li>
</ul></li>
<li><p><strong>Vue 3.x</strong>：</p>
<ul>
<li>使用 <code>Proxy</code> 实现响应式，依赖收集通过 <code>effect</code>
和 <code>ReactiveEffect</code> 实现。</li>
<li>当响应式数据被访问时，会触发 <code>track</code> 函数，将当前的
<code>effect</code>（类似于 Watcher）添加到依赖列表中。</li>
<li>当响应式数据被修改时，会触发 <code>trigger</code>
函数，通知所有依赖的 <code>effect</code> 更新。</li>
</ul></li>
</ul>
<h4 id="特点-7">特点：</h4>
<ul>
<li>这种 <code>watch</code> 是隐式的，Vue 会自动为 <code>data</code>
中的每个属性创建依赖关系。</li>
<li>主要用于模板渲染和计算属性的依赖收集。</li>
</ul>
<h3 id="2-计算属性的-watchercomputed-watcher">2. <strong>计算属性的
Watcher（Computed Watcher）</strong></h3>
<p>计算属性（<code>computed</code>）的实现依赖于一个特殊的
<code>Watcher</code>，称为
<strong>_computedWatcher</strong>。它的作用是监听计算属性所依赖的响应式数据，并根据依赖的变化重新计算值。</p>
<h4 id="源码实现-1">源码实现：</h4>
<ul>
<li><p><strong>Vue 2.x</strong>：</p>
<ul>
<li>在 <code>src/core/instance/state.js</code>
中，<code>initComputed</code> 函数会为每个计算属性创建一个
<code>Watcher</code> 实例。</li>
<li>这个 <code>Watcher</code> 的 <code>lazy</code> 属性为
<code>true</code>，表示它是惰性的，只有在需要时才会计算值。</li>
<li>计算属性的 <code>getter</code>
会被包装成一个函数，用于在访问计算属性时触发依赖收集。</li>
</ul></li>
<li><p><strong>Vue 3.x</strong>：</p>
<ul>
<li>计算属性的实现依赖于 <code>effect</code> 和 <code>computed</code>
函数。</li>
<li>在 <code>packages/reactivity/src/computed.ts</code>
中，<code>computed</code> 函数会创建一个 <code>ComputedRefImpl</code>
实例，内部使用 <code>effect</code> 监听依赖变化。</li>
</ul></li>
</ul>
<h4 id="特点-8">特点：</h4>
<ul>
<li>计算属性的 <code>Watcher</code>
是惰性的，只有在访问计算属性时才会计算值。</li>
<li>计算属性的值会被缓存，只有当依赖的响应式数据发生变化时才会重新计算。</li>
</ul>
<h3 id="3-开发者定义的-watcherwatch-api">3. <strong>开发者定义的
Watcher（Watch API）</strong></h3>
<p>开发者可以通过 <code>watch</code> 选项或 <code>$watch</code>
方法显式地监听响应式数据的变化。这种 <code>watch</code>
是通过创建一个普通的 <code>Watcher</code> 实例来实现的。</p>
<h4 id="源码实现-2">源码实现：</h4>
<ul>
<li><p><strong>Vue 2.x</strong>：</p>
<ul>
<li>在 <code>src/core/instance/state.js</code>
中，<code>initWatch</code> 函数会遍历 <code>watch</code>
选项，为每个监听属性创建一个 <code>Watcher</code> 实例。</li>
<li>这个 <code>Watcher</code> 的 <code>cb</code>
是用户定义的回调函数，当监听的数据发生变化时会被调用。</li>
<li>如果设置了
<code>deep: true</code>，则会递归遍历对象或数组，为每个子属性创建依赖。</li>
</ul></li>
<li><p><strong>Vue 3.x</strong>：</p>
<ul>
<li>在 <code>packages/runtime-core/src/apiWatch.ts</code>
中，<code>watch</code> 函数会创建一个 <code>ReactiveEffect</code>
实例，用于监听响应式数据的变化。</li>
<li>开发者可以通过 <code>watch</code>
函数监听单个或多个响应式数据，并指定回调函数。</li>
</ul></li>
</ul>
<h4 id="特点-9">特点：</h4>
<ul>
<li>这种 <code>watch</code> 是显式的，由开发者主动定义。</li>
<li>支持深度监听（<code>deep: true</code>）和立即执行（<code>immediate: true</code>）。</li>
</ul>
<h3 id="总结-2">总结</h3>
<p>从源码的角度来看，Vue 中的 <code>watch</code>
可以分为以下三种类型：</p>
<table>
<thead>
<tr>
<th>类型</th>
<th>实现机制</th>
<th>使用场景</th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>Data Watcher</strong></td>
<td>通过 <code>Object.defineProperty</code> 或 <code>Proxy</code>
实现依赖收集</td>
<td>模板渲染、计算属性的依赖收集</td>
</tr>
<tr>
<td><strong>Computed Watcher</strong></td>
<td>通过 <code>_computedWatcher</code> 实现惰性计算和缓存</td>
<td>计算属性的实现</td>
</tr>
<tr>
<td><strong>Watch API</strong></td>
<td>通过显式创建 <code>Watcher</code> 或 <code>ReactiveEffect</code>
实例实现监听</td>
<td>开发者主动监听响应式数据的变化</td>
</tr>
</tbody>
</table>
<p>这三种 <code>watch</code> 共同构成了 Vue
的响应式系统，分别用于不同的场景和需求。</p>
