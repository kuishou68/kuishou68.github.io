<p>前言：<strong>KeepAlive</strong>
这个词相信对你并不陌生，没错，正是借鉴了HTTP协议，在HTTP中的keep-alive又称<strong>HTTP持久连接</strong>(HTTP
persistent connection)，作用是允许多个请求或响应共用一个TCP链接。
http中的KeepaAlive可以避免链接频繁创建/销毁，在Vue中的<a
href="https://cn.vuejs.org/guide/built-ins/keep-alive.html#keepalive">keepalive</a>同样也是为了避免一个组件频繁创建/重建。

<h3 id="11-解决了什么问题">1.1 解决了什么问题？</h3>
<p>可以看到，如果频繁的切换 currTab
，会导致不停地卸载并重建对应的tab组件，为了避免不必要的开销，可以使用keepalive包裹，同时还可以通过 <code>include</code> 和 <code>exclude</code> prop
来定制缓存行为，会根据组件的 <a
href="https://cn.vuejs.org/api/options-misc.html#name"><code>name</code></a> 选项进行匹配，另外，<code>max</code>
最大缓存实例数，缓存的实例数量即将超过指定的那个最大数量，则最久没有被访问的缓存实例将被销毁，以便为新的实例腾出空间。</p>
<h3 id="12-实现原理">1.2 实现原理</h3>
<p>其实keepAlive的本质是<strong>缓存管理</strong>，再加上特殊的挂载/卸载逻辑。首先，KeepAlive
组件的实现需要渲染层的支持，而KeepAlive的组件在卸载时，我们也不能真正的将其卸载否则无法维持组件当前的状态了。通过观察源码发现，被KeepAlive的组件从原容器搬运到了另一个隐藏的容器中，实现了“假卸载”；当隐藏容器中的组件被”挂载“的时候，再从隐藏容器中搬运到原容器。这个过程就是
<a
href="https://cn.vuejs.org/api/options-lifecycle.html#activated"><code>activated</code></a>
和 <a
href="https://cn.vuejs.org/api/options-lifecycle.html#deactivated"><code>deactivated</code></a>。</p>
<h3 id="13-include-和-exclude">1.3 include 和 exclude</h3>
<p>默认情况下，KeepAlive组件会对所有”内部组件“进行缓存，但有时候期望只缓存特定组件，<code>include</code>
用来显示的配置应该被缓存的组件，<code>exclude</code>用来显示的配置不应该缓存的组件。源码中会根据”内部组件“的名称name进行匹配，判断是否进行缓存。</p>
<h3 id="14-缓存管理">1.4 缓存管理</h3>
<p>这也是KeepAlive的核心，其内部通过一个Map对象来实现对组件的缓存，该Map的key是组件选项对象，即
<code>vnode.type</code>属性的值，而Map的Value是用于描述组件的vnode对象。</p>

<h3 id="15-可能出现的问题">1.5 可能出现的问题</h3>
<p>上面代码看似没毛病，但是有个细节，如果缓存不存在的时候，总是会设置新的缓存，这会导致缓存不断地增加。为了解决这个问题，必须设置一个缓存阈值，当缓存数量超过指定阈值时将其进行修剪。那么你可能会问，进行修剪的依据是什么呢？vue中当前采用的策略是
”最新一次访问“，结合<code>max</code>最大缓存实例数，把当前访问（或修剪）的组件作为最新一次渲染的组件，并且该组件在缓存修剪过程中是安全且不会被修剪的。</p>
<h2 id="2-teleport">2 Teleport</h2>
<h3 id="21-解决了什么问题">2.1 解决了什么问题？</h3>
<p>通常情况下，在将虚拟dom渲染成真实dom时，最终渲染出来的真实dom层级结果与虚拟dom层级结构一致。</p>

<p>上面这段代码中，<code>&lt;Overlay /&gt;</code>组件的内容会被渲染到id位main的div标签下，假设<code>&lt;Overlay /&gt;</code>是一个”蒙层“组件，并要求蒙层能遮挡页面上的任何元素，也就是要求<code>&lt;Overlay /&gt;</code>的z-index层级最高来实现。但问题是，如果<code>&lt;Overlay /&gt;</code>组件的内容无法跨域dom层级渲染，就无法实现这个目标。
于是就出现了<a
href="https://cn.vuejs.org/guide/built-ins/teleport.html#teleport">Teleport</a>内置组件，该组件<strong>可以将指定的内容渲染到特定容器中，并且不受dom层级限制。</strong></p>

_isTeleport 标识判断是否为 Teleport组件，如果是，则直接调用
<code>process</code>
函数，将渲染控制权完全交接出去。通过判断分离出来的dom是否存在，来决定执行挂载还是更新，如果是挂载，则根据<code>props.to</code>的属性来去的真正的挂载点。<code>to</code> 的值可以是一个
CSS 选择器字符串，也可以是一个 DOM 元素对象。</p>

<h3 id="32-实现原理">3.2 实现原理</h3>
<p>实现逻辑其实和元素DOM很相似，不过<a

leave等。</p>

<p>在dom挂载之前，调用
<code>transition.beforeEnter</code>钩子；挂载之后，调用
<code>transition.enter</code>，并且两个钩子函数都接收需要过渡的dom元素对象作为的个参数；卸载元素则调用
