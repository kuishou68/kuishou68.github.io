<p><strong>1、数据响应式原理</strong></p>
</blockquote>
<blockquote>
<p><strong>1.1、MVVM是什么？</strong><br />
简单来说，就是数据变了，视图也会跟着变，首先你得定义一个带有{{
}}的模板Model，当数据中的值变化了，视图View就会跟着变化；视图模型View-model是模板Model和视图View之间的桥梁，Vue属于非侵入式，React和小程序就是侵入式（数据变化的时候需要调用提前写好的API）</p>
</blockquote>
<pre><code>// Vue数据变化，非侵入式
this.a ++
// React、小程序数据变化，侵入式
this.setState({
    a: this.state.a + 1
});</code></pre>
<blockquote>
<p><strong>1.2、数据响应式的中心思想？</strong><br />
通过重写数据的get和set属性方法，让数据在被渲染时，把所有用到自己的订阅者，存放在自己的订阅者列表中；当数据发生改变时，该方法通知所有订阅了自己的订阅者，达到重新渲染的目的。</p>
</blockquote>

</blockquote>
<blockquote>
<p>答：以Array.prototype为原型，创建了一个arrayMethods的对象，用一个非常强硬的手段，Object.setPrototypeOf()让数组的_
<em>proto</em>
_强制指向arrayMethods，这样就可以调用新的改写的7个方法。</p>

<blockquote>
<p><strong>5、什么是依赖？</strong><br />
需要用到数据的递归就是<strong>依赖</strong>，在getter中<strong>收集</strong>依赖，在setter中<strong>触发</strong>依赖；收集依赖的代码封装成Dep类，每个Observer的实例都有一个Dep的实例；Watcher是一个中介，数据发生变化时通过watcher中转，通知组件。</p>

<pre><code>&lt;script&gt;
    // 深层套娃
    var o = {
        a: {
            b: {
                c: {
                    d: 68
                }
            }
        }
    }
    var str = &#39;a.b.c.d&#39;;
    function parsePath(str){
        // 根据 . 来进行拆分
        var segments = str.split(&#39;.&#39;);
        // 返回接收对象的函数
        return(obj) =&gt; {
            // 遍历接收的函数
            for(let i = 0; i &lt; segments.length; i++){
                // 判断obj存不存在
                if(!obj) return;
                // 一层一层的剥开 o 的心
                obj = obj[segments[i]]
            }
            // 高阶函数，函数内部返回一个函数
            return obj;
        }
    }
    // 调用一下
    var fn = parsePath(str);
    var v = fn(o);
    console.log(v);
