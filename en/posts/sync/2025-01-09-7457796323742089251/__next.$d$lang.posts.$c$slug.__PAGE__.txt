<h3 id="1proxy">1.<a
href="https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Proxy">Proxy</a></h3>
<p>MDN上的解释：用于创建一个对象的代理，可以拦截和重定义一个对象的基本方法，那么什么叫做对象的基本方法呢？先看下面对象的基本操作。</p>
<div class="sourceCode" id="cb1"><pre class="sourceCode js"><code class="sourceCode javascript"><span id="cb1-1"><a href="#cb1-1" aria-hidden="true" tabindex="-1"></a><span class="kw">let</span> obj <span class="op">=</span> {}<span class="op">;</span></span>
<span id="cb1-2"><a href="#cb1-2" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb1-3"><a href="#cb1-3" aria-hidden="true" tabindex="-1"></a><span class="co">// 添加一个属性</span></span>
<span id="cb1-4"><a href="#cb1-4" aria-hidden="true" tabindex="-1"></a>obj<span class="op">.</span><span class="at">a</span> <span class="op">=</span> <span class="dv">3</span><span class="op">;</span> <span class="co">// 相当于调用了[[Set]]</span></span>
<span id="cb1-5"><a href="#cb1-5" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb1-6"><a href="#cb1-6" aria-hidden="true" tabindex="-1"></a><span class="co">// 读取一个属性</span></span>
<span id="cb1-7"><a href="#cb1-7" aria-hidden="true" tabindex="-1"></a>obj<span class="op">.</span><span class="at">a</span><span class="op">;</span> <span class="co">// 相当于调用了[[Get]]</span></span>
<span id="cb1-8"><a href="#cb1-8" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb1-9"><a href="#cb1-9" aria-hidden="true" tabindex="-1"></a><span class="co">// 设置对象的原型</span></span>
<span id="cb1-10"><a href="#cb1-10" aria-hidden="true" tabindex="-1"></a><span class="bu">Object</span><span class="op">.</span><span class="fu">setPrototypeOf</span>(obj<span class="op">,</span> {<span class="dt">a</span><span class="op">:</span><span class="dv">3</span>})<span class="op">;</span> <span class="co">// 相当于调用了 [[SetPrototypeOf]]</span></span>
<span id="cb1-11"><a href="#cb1-11" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb1-12"><a href="#cb1-12" aria-hidden="true" tabindex="-1"></a><span class="co">// 读取对象的原型</span></span>
<span id="cb1-13"><a href="#cb1-13" aria-hidden="true" tabindex="-1"></a><span class="bu">Object</span><span class="op">.</span><span class="fu">getPrototypeOf</span>(obj)<span class="op">;</span> <span class="co">// 相当于调用了 [[GetPrototypeOf]]</span></span>
<span id="cb1-14"><a href="#cb1-14" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb1-15"><a href="#cb1-15" aria-hidden="true" tabindex="-1"></a><span class="co">// 遍历对象的属性，相当于调用了对象的 [[getOwnPropertyKeys]]</span></span>
<span id="cb1-16"><a href="#cb1-16" aria-hidden="true" tabindex="-1"></a><span class="cf">for</span>(<span class="kw">const</span> key <span class="kw">in</span> obj){</span>
<span id="cb1-17"><a href="#cb1-17" aria-hidden="true" tabindex="-1"></a>    </span>
<span id="cb1-18"><a href="#cb1-18" aria-hidden="true" tabindex="-1"></a>}</span></code></pre></div>
<p>上面的基本操作其实本质上会转换成调用对象的内部方法，比如<code>obj.a=3</code>
会调用对象的内部方法
[[Set]]，这方法在外面是访问不到的，只有在内部才能访问到。也就是说，平时我们js语法层面，或者是API层面，去对对象进行基本操作，最终在底层都是在调用对象的内部方法。换句话说，你对对象的基本操作，是逃不过这个基本方法的。在<a
href="https://ecma-international.org/wp-content/uploads/ECMA-262_15th_edition_june_2024.pdf">ES262</a>文档里面可以看到所有的<a
href="https://ecma-international.org/wp-content/uploads/ECMA-262_15th_edition_june_2024.pdf">内部方法</a>：</p>
<p><img
src="https://p6-xtjj-sign.byteimg.com/tos-cn-i-73owjymdk6/28d93146bf6645b0bb6857618d7a06e9~tplv-73owjymdk6-jj-mark-v1:0:0:0:0:5o6Y6YeR5oqA5pyv56S-5Yy6IEAgQ29jb29uQnJlYWs=:q75.awebp?rk3s=f64ab15b&amp;x-expires=1773325746&amp;x-signature=ljnAfhB4gPUXBjii5XuYUvJPHu0%3D"
alt="image.png" /></p>
<p>这里的内部方法，就是Proxy解释中说的基本方法。数组本质上也是一个对象，区别于<code>常规对象</code>，普遍把数组叫做<code>异质对象</code>，常规对象和异质对象的不同体现在[[DefineOwnProperty]]这个内部方法上。Proxy重定义对象基本方法的同时也可以对这些基本方法进行拦截，换句话说，Proxy可以拦截对象的一些操作。你可能会问，函数也能拦截吗？答案是可以的函数在内部也是一个对象，只不过多了两个内部方法<code>[[Call]]</code>和<code>[[Construct]]</code>，你通过new调用而已。</p>
<h3 id="2defineproperty">2.<a
href="https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Object/defineProperty">DefineProperty</a></h3>
<p>MDN解释：会直接在一个对象上定义一个新属性，或修改其现有属性，并返回此对象。</p>
<div class="sourceCode" id="cb2"><pre class="sourceCode js"><code class="sourceCode javascript"><span id="cb2-1"><a href="#cb2-1" aria-hidden="true" tabindex="-1"></a><span class="kw">const</span> arr <span class="op">=</span> [<span class="dv">1</span><span class="op">,</span> <span class="dv">2</span><span class="op">,</span> <span class="dv">3</span>]<span class="op">;</span></span>
<span id="cb2-2"><a href="#cb2-2" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb2-3"><a href="#cb2-3" aria-hidden="true" tabindex="-1"></a><span class="co">// 往数组里面加一项，会导致数组的长度也变化</span></span>
<span id="cb2-4"><a href="#cb2-4" aria-hidden="true" tabindex="-1"></a>arr<span class="op">.</span><span class="fu">push</span>(<span class="dv">1</span>)<span class="op">;</span> </span>

<p>这个时候你会发现，vue2没办法直接拦截数组原型，只能在数组本身查找数组原型的过程中重写push、shift、unshift这些方法。更改数组的原型链，你在vue2中调用的push方法，其实调用的是重写过后的push方法，这样就可以进行拦截。</p>

<p><strong>总结：</strong>
Proxy拦截的是针对对象的所有基本方法，他的拦截是全面的，没有任何遗漏；而defineProperty只是众多基本方法其中之一，作用是对现有对象的拦截，拦截面是不全面的。</p>
